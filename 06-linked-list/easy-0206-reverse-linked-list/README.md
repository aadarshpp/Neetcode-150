# 206. Reverse Linked List

## Difficulty
Easy

## Complexity
- Time: O(n)
- Space: O(1)

## Approach
Traverse the list from start to end using three variables: prev, curr, and next.Use next to keep track of the next node so we don’t lose it, and use prev and curr to reverse the direction of the links.
